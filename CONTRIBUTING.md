# Contributing to FaceCast

Thanks for your interest in contributing.

## Before You Start

- Open an issue for bugs, regressions, or feature proposals when possible.
- Keep pull requests focused and small.
- Avoid mixing refactors with feature changes unless they are required.

## Development

1. Open `FaceCast.xcodeproj` in Xcode 15 or later.
2. Build and run on macOS 13 or later.
3. Grant the required permissions for screen, camera, and microphone access.

## Pull Requests

- Describe the user-facing change clearly.
- Include validation steps.
- Update docs when behavior or setup changes.
- Do not commit build artifacts, local Xcode state, or packaging output.

## Code Style

- Follow the existing Swift and SwiftUI patterns in the project.
- Prefer focused changes over broad rewrites.
- Keep UI and capture-pipeline changes easy to review.
